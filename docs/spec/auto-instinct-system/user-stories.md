## エピック1: セッション操作の自動記録

### ストーリー 1.1: ツール使用の自動ログ 🔵

**信頼性**: 🔵 *ヒアリング確認済み*

**私は** 開発者として
**Claude Code を使って開発しているだけで、どのツールをどの順番で使ったか自動的に記録したい**
**そうすることで** 意識せずに自分の操作パターンを蓄積できる

**関連要件**: REQ-001, REQ-002, REQ-003

**詳細シナリオ**:
1. Claude Code で Bash・Edit・Write などのツールを実行する
2. PostToolUse Hook が自動で `.claude/session-log.jsonl` に追記する
3. 開発者は何も意識しなくてよい

**前提条件**:
- `.claude/settings.local.json` に PostToolUse hook が設定済み

**優先度**: Must Have

---

### ストーリー 1.2: エラー→修正パターンの記録 🔵

**信頼性**: 🔵 *ヒアリング確認済み*

**私は** 開発者として
**コマンドが失敗してリトライした操作の流れを記録したい**
**そうすることで** 同じ失敗を繰り返す前に警告が得られる**

**関連要件**: REQ-001, REQ-011

**詳細シナリオ**:
1. Bash コマンドが失敗（exit code != 0）する
2. 続いて修正コマンドが成功する
3. この「失敗→成功」の連鎖がログに記録される
4. Stop 時に「エラー→修正パターン」として抽出される

**前提条件**:
- セッションログに outcome フィールドが含まれる

**優先度**: Must Have

## エピック2: セッション終了時の自動分析

### ストーリー 2.1: instinct ファイルの自動生成 🔵

**信頼性**: 🔵 *ヒアリング確認済み*

**私は** 開発者として
**セッションを終えるだけで、今日の操作パターンが instinct ファイルに保存されてほしい**
**そうすることで** パターンが自然に蓄積し、プロジェクトの知識として残る

**関連要件**: REQ-010, REQ-011, REQ-012, REQ-013

**詳細シナリオ**:
1. 開発セッションを終了する（Claude Code が Stop）
2. `instinct-on-stop.sh` が自動起動する
3. `session-log.jsonl` を AI（claude -p）で分析する
4. 抽出されたパターンが `.claude/instincts/{slug}.md` に保存される
5. 既存の instinct と同一パターンなら occurrence_count が増加する
6. セッションログがリセットされる

**前提条件**:
- session-log.jsonl にデータが存在する

**優先度**: Must Have

---

### ストーリー 2.2: 信頼度の自動昇格 🔵

**信頼性**: 🔵 *ヒアリング確認済み*

**私は** 開発者として
**繰り返し現れるパターンが自動的に高信頼度に昇格してほしい**
**そうすることで** ノイズと本物のパターンが自然に分離される

**関連要件**: REQ-020, REQ-021, REQ-022

**詳細シナリオ**:
1. 初回検出: confidence = low
2. 2〜4回検出: confidence = medium に自動昇格
3. 5回以上検出: confidence = high に自動昇格

**前提条件**:
- instinct ファイルの occurrence_count が正確に更新される

**優先度**: Must Have

## エピック3: diary-sync との統合

### ストーリー 3.1: instinct を CLAUDE.md 統合候補に昇格 🔵

**信頼性**: 🔵 *ヒアリング確認済み*

**私は** 開発者として
**週1の `/diary-sync` 実行時に、高信頼度の instinct も CLAUDE.md への統合候補として提案されてほしい**
**そうすることで** 手動で learnings.md を確認しなくても、パターンが CLAUDE.md に反映される

**関連要件**: REQ-030, REQ-031, REQ-032

**詳細シナリオ**:
1. `/diary-sync` を実行する
2. `learnings.md` に加えて `.claude/instincts/` も読み込まれる
3. confidence `high` の instinct が CLAUDE.md 統合候補として優先提案される
4. 承認した instinct に `promoted: true` フラグが付与される

**前提条件**:
- `.claude/instincts/` に confidence high のファイルが存在する

**優先度**: Must Have

---

### ストーリー 3.2: /evolve で instinct を管理 🟡

**信頼性**: 🟡 *ECC 設計から妥当な推測*

**私は** 開発者として
**蓄積された instinct の一覧を確認・管理したい**
**そうすることで** 不要なパターンを削除したり、手動で信頼度を変更したりできる

**関連要件**: REQ-040, REQ-041

**詳細シナリオ**:
1. `/evolve` を実行する
2. `.claude/instincts/` の全ファイルが信頼度付きで一覧表示される
3. 不要なパターンを選んで削除できる

**優先度**: Should Have
