# ADR 0003: タスクステータスを todo → doing → done → todo のサイクルにする

## ステータス
採用済み

## 背景
タスク管理でよくある Kanban ボード方式（ドラッグ＆ドロップ）は実装コストが高く、モバイルでも使いにくい。

## 決定
ステータスをボタン1タップで循環させる：`todo → doing → done → todo`

`TaskStatusButton` コンポーネントが現在のステータスに応じて次のステータスへ遷移する action を受け取る。

## 理由
- 実装がシンプル
- モバイルでも操作しやすい
- 個人利用では並列ワークフローより直線的なサイクルで十分

## 影響
- ステータスの逆戻し（done → doing）も循環で行う
- `updateTaskStatus` 関数がステータス遷移ロジックを担う
