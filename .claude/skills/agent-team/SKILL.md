# エージェントチーム

複数の Claude インスタンスがチームとして協調動作する（実験的機能）。

## subagent との使い分け

| 状況 | 使うもの |
|---|---|
| コードレビュー・DB調査など焦点を絞った調査 | subagent |
| チームメイト同士が発見を共有・議論する必要がある | エージェントチーム |
| 複数の仮説を並列でデバッグ | エージェントチーム |
| フロントエンド・バックエンド・テストを同時に実装 | エージェントチーム |

## このプロジェクトでの活用例

### 並列コードレビュー
```
エージェントチームを作成して PR をレビューして。
3人のレビュアーを立てる：
- セキュリティ（XSS, SQLインジェクション, 環境変数）
- Next.js パターン（Server/Client Component の使い分け）
- ダークモード対応漏れ・UI 一貫性
```

### 競合する仮説でデバッグ
```
[バグの症状を説明]。
5人のチームメイトで異なる仮説を並列調査して。
互いの理論を反証し合い、consensus をまとめて。
```

### 大きな機能の並列実装
```
タグ機能を実装するエージェントチームを作って。
- チームメイトA: DBマイグレーション + queries.ts
- チームメイトB: Server Action + フォームUI
- チームメイトC: フィルタリングUI
実装前にプラン承認を要求して。
```

## 注意事項

- トークン消費が多い（チームメイト数に比例）
- 同じファイルを複数チームメイトが編集すると上書き競合が起きる
- チームメイトは3〜5人が適切
- 完了後は必ず `Clean up the team` を実行
