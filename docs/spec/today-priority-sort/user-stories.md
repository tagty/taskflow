# 今日ビューの判断支援強化 ユーザストーリー

## エピック1: 優先度順で今日の作業を把握する

### ストーリー 1.1: 最優先タスクを一目で把握する 🔵

**私は** 開発者 **として**
**Today ページを開いたとき priority 順に並んだタスク一覧を見たい**
**そうすることで** 今日何から着手するかを迷わず判断できる

**関連要件**: REQ-001, REQ-002

**詳細シナリオ**:
1. `/today` を開く
2. priority=1 のタスクが最上位に表示される
3. priority=2, 3... と続き、priority=null のタスクが末尾に並ぶ

**優先度**: Must Have

## エピック2: 今日の作業量を把握して過負荷を防ぐ

### ストーリー 2.1: 見積時間の合計を確認する 🔵

**私は** 開発者 **として**
**今日のタスクの見積時間合計を確認したい**
**そうすることで** 今日の作業量が現実的かを素早く判断できる

**関連要件**: REQ-003, REQ-101, REQ-102

**詳細シナリオ**:
1. `/today` を開く
2. 「今日のタスク」セクションに「合計 Xh Ym」と表示される
3. estimate_minutes が null のタスクは合計に含めない

**優先度**: Must Have

### ストーリー 2.2: 過負荷を警告で気づく 🔵

**私は** 開発者 **として**
**合計見積時間が 8h を超えたときに警告を受けたい**
**そうすることで** 詰め込みすぎに気づき、タスクを翌日以降に移せる

**関連要件**: REQ-004

**詳細シナリオ**:
1. 合計 estimate_minutes が 480 を超える
2. 合計表示の横または下に警告インジケーターが表示される
3. 警告は視覚的に目立つ（amber 系の色）

**優先度**: Must Have
