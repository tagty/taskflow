# セルフレビュー（5エージェント並列）

PR 作成前に **5つの専門エージェントを並列起動** してコードをレビューする。
各エージェントは独立した観点で調査し、結果をオーケストレーターが統合する。

## 実行方法

```
/self-review
```

## エージェント構成

Agent ツールで以下の5エージェントを **同時に** 起動する：

### Agent 1 — 規約番人
- Tailwind CSS に `dark:` クラスが付いているか
- `supabaseAdmin` がクライアント側で使われていないか
- クエリが `queries.ts` 以外に書かれていないか
- `docs/adr/` の決定事項に反していないか
- コミットメッセージが日本語か

### Agent 2 — セキュリティ監査
- `supabaseAdmin`（secret key）のクライアント露出がないか
- 環境変数が `NEXT_PUBLIC_` 以外でクライアントに漏れていないか
- XSS・SQLインジェクションのリスクがないか
- Server Actions 以外からの DB write がないか

### Agent 3 — 型・品質検査
- 型エラー・型 `any` の使用がないか
- 未使用の変数・インポートがないか
- エラーハンドリングが適切か（`if (error) throw error` パターン）
- null/undefined の取り扱いが安全か

### Agent 4 — テスト検証
- 新しい関数・ロジックにテストがあるか
- テストが実際の動作を検証しているか（モックで誤魔化していないか）
- `npm run verify` が通るか
- 追加すべきテストケースの提案

### Agent 5 — 要件照合
- Issue / PR の要件をすべて実装しているか
- `docs/spec/` の要件定義書との整合性
- 実装漏れ・過剰実装がないか
- UI の dark モード対応が視覚的に一貫しているか

## 実行手順

1. `git diff main...HEAD` で変更差分を取得
2. 上記5エージェントを Agent ツールで **並列起動**（subagent_type: Explore）
3. 全エージェントの結果を収集・統合
4. 問題点を重大度（🔴 MUST / 🟡 SHOULD / 🔵 INFO）でソート
5. 🔴 MUST の問題がある場合は修正してから PR 作成

## 出力フォーマット

```
## セルフレビュー結果

### Agent 1 — 規約番人 ✅ / ⚠️
- ...

### Agent 2 — セキュリティ監査 ✅ / ⚠️
- ...

### Agent 3 — 型・品質検査 ✅ / ⚠️
- ...

### Agent 4 — テスト検証 ✅ / ⚠️
- ...

### Agent 5 — 要件照合 ✅ / ⚠️
- ...

## 統合判定

🔴 MUST 修正:
- [ ] ...

🟡 SHOULD 修正:
- [ ] ...

🔵 INFO（任意）:
- ...

## 判定: ✅ PR 作成可 / ⚠️ 要修正
```

## 注意事項

- 5エージェント並列起動のためトークン消費が多い
- 修正が必要な場合は修正 → `npm run verify` → 再レビューのループ
- 全 🔴 が解消されたら PR 作成に進む
