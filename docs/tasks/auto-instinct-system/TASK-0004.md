## タスク概要

Stop hook として動作する `instinct-on-stop.sh` を実装する。セッション終了時に `session-log.jsonl` を `claude -p` で AI 分析し、3種のパターン（ツール使用・コマンド・エラー→修正）を `.claude/instincts/{slug}.md` に抽出・保存する。既存パターンとの意味的類似度を判定し、occurrence_count と confidence を更新する。分析後は session-log.jsonl をリセットする。

## 実装詳細

### 1. session-log.jsonl の空チェック 🔵

**信頼性**: 🔵 *NFR-020・REQ-010 より*

```bash
LOG_FILE="$(pwd)/.claude/session-log.jsonl"
INSTINCTS_DIR="$(pwd)/.claude/instincts"

if [ ! -f "$LOG_FILE" ] || [ ! -s "$LOG_FILE" ]; then
  exit 0  # サイレント終了
fi
```

**実装ファイル**: `.claude/hooks/instinct-on-stop.sh`

### 2. instincts/ ディレクトリの自動作成 🔵

**信頼性**: 🔵 *EDGE-001 より*

```bash
mkdir -p "$INSTINCTS_DIR"
```

### 3. ログのサイズ制限（直近500行） 🔴

**信頼性**: 🔴 *EDGE-010 から推測*

1000行を超える場合は直近500行のみを分析対象とする。

```bash
LINE_COUNT=$(wc -l < "$LOG_FILE")
if [ "$LINE_COUNT" -gt 1000 ]; then
  SESSION_LOG=$(tail -500 "$LOG_FILE")
else
  SESSION_LOG=$(cat "$LOG_FILE")
fi
```

### 4. 既存 instinct ファイル一覧の取得 🔵

**信頼性**: 🔵 *REQ-023・architecture.md より*

```bash
EXISTING_INSTINCTS=""
if ls "$INSTINCTS_DIR"/*.md 2>/dev/null | head -1 > /dev/null; then
  EXISTING_INSTINCTS=$(cat "$INSTINCTS_DIR"/*.md 2>/dev/null)
fi
```

### 5. claude -p での AI 分析 🔵

**信頼性**: 🔵 *file-formats.md のプロンプト仕様より*

```bash
claude -p "
以下のセッションログを分析して、プロジェクト固有のパターンを抽出してください。

## セッションログ
$SESSION_LOG

## 既存の instinct ファイル一覧
$EXISTING_INSTINCTS

## 抽出するパターン種別
1. ツール使用パターン: ツールの連鎖順序（例: Glob→Read→Edit）
2. コマンドパターン: 繰り返し実行されるbashコマンドの組み合わせ
3. エラー→修正パターン: エラー発生→修正成功の連鎖

## instinct ファイルのフォーマット
\`\`\`yaml
---
slug: {pattern-slug}
title: {短い説明}
type: tool-usage | command | error-fix
confidence: low | medium | high
occurrence_count: {数値}
last_seen: {YYYY-MM-DD}
promoted: false
---

## パターン説明
{説明}

## 観測例
{例}

## 示唆
{示唆}
\`\`\`

## 出力ルール
- 既存 instinct と意味的に類似するパターンは、そのファイルを更新する（occurrence_count++ と last_seen 更新）
- confidence は occurrence_count に基づく: 1→low, 2〜4→medium, 5以上→high
- 新規パターンは confidence: low で新しいファイルを作成する
- 意味のない1回限りの操作はパターンとして記録しない
- 各ファイルは $INSTINCTS_DIR/{slug}.md に保存する

ファイルを更新・作成してください。
" --allowedTools "Read,Edit,Write" 2>/dev/null
```

## 注意事項

- `--allowedTools "Read,Edit,Write"` を指定して claude -p の権限を最小化すること
- Stop hook は本来の開発フローをブロックしてはならない（常に exit 0）

- session-log.jsonl のパスは絶対パスで指定すること
