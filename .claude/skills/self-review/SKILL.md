# セルフレビュー

PR 作成前に以下の観点を **並列で** チェックし、問題があれば修正する。

## 実行方法

```
/self-review
```

## チェック観点（並列実行）

以下を subagent で並列チェックする：

### 1. 規約チェック
- Tailwind CSS に `dark:` クラスが付いているか
- `supabaseAdmin` がクライアント側で使われていないか
- クエリが `queries.ts` 以外に書かれていないか
- コミットメッセージが日本語か
- `docs/adr/` の決定事項に反していないか

### 2. コード品質チェック
- 型エラー・型 `any` の使用がないか
- 未使用の変数・インポートがないか
- エラーハンドリングが適切か（`if (error) throw error` パターン）
- セキュリティ上の問題（XSS, SQLインジェクション, 環境変数の漏洩）がないか

### 3. テスト確認
- 新しい関数・ロジックにテストがあるか
- `npm run verify` が通るか

### 4. 要件確認
- Issue / PR の要件をすべて実装しているか
- 実装漏れ・過剰実装がないか

## 出力フォーマット

各観点ごとに結果を報告し、問題点があれば修正案を提示する。

```
## セルフレビュー結果

### 1. 規約チェック ✅ / ⚠️
- ...

### 2. コード品質チェック ✅ / ⚠️
- ...

### 3. テスト確認 ✅ / ⚠️
- ...

### 4. 要件確認 ✅ / ⚠️
- ...

## 修正が必要な項目
- [ ] ...
```

問題がなければ PR 作成に進む。
