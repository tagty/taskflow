## 実装詳細

### 1. 環境変数の取得 🔵

**信頼性**: 🔵 *REQ-001・dataflow.md より*

Claude Code の PostToolUse hook では以下の環境変数が利用可能:
- `CLAUDE_TOOL_NAME`: ツール名
- `CLAUDE_TOOL_INPUT`: ツールへの入力（JSON）
- `CLAUDE_TOOL_RESULT`: ツールの実行結果

**実装ファイル**: `.claude/hooks/instinct-logger.sh`

### 2. input_summary の生成（最大200文字） 🔵

**信頼性**: 🔵 *REQ-002・file-formats.md より*

`CLAUDE_TOOL_INPUT` から主要な情報を抽出して200文字以内に切り詰める。

```bash
INPUT_SUMMARY=$(echo "$CLAUDE_TOOL_INPUT" | head -c 200)
```

### 3. 機密情報のマスキング 🔵

**信頼性**: 🔵 *NFR-010・file-formats.md より*

以下のパターンをマスキング:

```bash
INPUT_SUMMARY=$(echo "$INPUT_SUMMARY" | \
  sed 's/password=[^ ]*/password=[REDACTED]/gi' | \
  sed 's/secret=[^ ]*/secret=[REDACTED]/gi' | \
  sed 's/key=[^ ]*/key=[REDACTED]/gi' | \
  sed 's/Bearer [^ ]*/Bearer [REDACTED]/g' | \
  sed 's/NEXT_PUBLIC_SUPABASE_[A-Z_]*=[^ ]*/[REDACTED]/g')
```

### 4. outcome の判定 🔵

**信頼性**: 🔵 *REQ-001・dataflow.md より*

`CLAUDE_TOOL_RESULT` の内容からエラー判定を行う。

```bash
if echo "$CLAUDE_TOOL_RESULT" | grep -q '"isError":true'; then
  OUTCOME="error"
  ERROR_SUMMARY=$(echo "$CLAUDE_TOOL_RESULT" | head -c 100)
else
  OUTCOME="success"
  ERROR_SUMMARY="null"
fi
```

### 5. session_id の生成 🟡

**信頼性**: 🟡 *REQ-002 から妥当な推測*

セッション識別子として、プロセスIDやタイムスタンプベースの文字列を使用する。

```bash
SESSION_ID="${PPID:-$$}"
```

### 6. JSONL への追記 🔵

**信頼性**: 🔵 *REQ-001・file-formats.md より*

```bash
LOG_FILE="$(pwd)/.claude/session-log.jsonl"
TIMESTAMP=$(date -u +"%Y-%m-%dT%H:%M:%SZ")

echo "{\"timestamp\":\"$TIMESTAMP\",\"tool\":\"$TOOL_NAME\",\"input_summary\":\"$INPUT_SUMMARY\",\"outcome\":\"$OUTCOME\",\"error_summary\":$ERROR_SUMMARY_JSON,\"session_id\":\"$SESSION_ID\"}" >> "$LOG_FILE"
```

## 単体テスト要件

### テストケース1: Bash 成功時のログ追記 🔵

**信頼性**: 🔵 *TC-001-01 より*

**Given**: session-log.jsonl が空
**When**: CLAUDE_TOOL_NAME=Bash、成功結果で instinct-logger.sh を実行
**Then**: `{"tool":"Bash","outcome":"success",...}` が session-log.jsonl に追記される

**テスト方法**:
```bash

cat .claude/session-log.jsonl
```
