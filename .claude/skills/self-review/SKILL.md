# セルフレビュー（5エージェント並列 + 自動修正）

PR 作成前に **5つの専門エージェントを並列起動** してコードをレビューし、
その後 **自動修正フェーズ** で適切なコメントのみを選別して修正する。

## 実行フロー

```
Phase 1: 5エージェント並列レビュー
    ↓
Phase 2: コメント選別・自動修正（fix-review-comments）
    ↓
Phase 3: npm run verify で動作確認
    ↓
PR 作成可
```

## Phase 1: エージェント構成

Agent ツールで以下の5エージェントを **同時に** 起動する：

### Agent 1 — 規約番人
- Tailwind CSS に `dark:` クラスが付いているか
- `supabaseAdmin` がクライアント側で使われていないか
- クエリが `queries.ts` 以外に書かれていないか
- `docs/adr/` の決定事項に反していないか
- コミットメッセージが日本語か

### Agent 2 — セキュリティ監査
- `supabaseAdmin`（secret key）のクライアント露出がないか
- 環境変数が `NEXT_PUBLIC_` 以外でクライアントに漏れていないか
- XSS・SQLインジェクションのリスクがないか
- Server Actions 以外からの DB write がないか

### Agent 3 — 型・品質・設計検査
- 型エラー・型 `any` の使用がないか
- 未使用の変数・インポートがないか
- エラーハンドリングが適切か（`if (error) throw error` パターン）
- null/undefined の取り扱いが安全か
- `.claude/rules/design-principles.md` の設計原則に違反していないか（SRP・DRY・YAGNI・依存の方向）
- `.claude/rules/functional-style.md` のスタイルガイドに従っているか（ビジネスロジック層・Server Actions は必須、コンポーネント層は推奨）

### Agent 4 — テスト検証
- 新しい関数・ロジックにテストがあるか
- テストが実際の動作を検証しているか（モックで誤魔化していないか）
- `npm run verify` が通るか
- 追加すべきテストケースの提案

### Agent 5 — 要件照合
- Issue / PR の要件をすべて実装しているか
- `docs/spec/` の要件定義書との整合性
- 実装漏れ・過剰実装がないか
- UI の dark モード対応が視覚的に一貫しているか

## Phase 2: コメント選別・自動修正（fix-review-comments）

5エージェントの結果を収集後、**各コメントを批判的に評価**し、適切なものだけを修正する。

### 選別ルール

**修正する（✅ 適用）:**
- ロジックの明確化・一貫性の向上
- 明らかなバグ・型エラー
- セキュリティ上のリスク
- 規約違反（dark: クラス漏れ、queries.ts 外のクエリなど）

**修正しない（❌ 却下）:**
- 意図的な設計判断に反するもの（理由を説明）
- すでに別のコメントで対応済みのもの
- 要件の範囲外・将来の改善提案
- スタイルの好みレベルの指摘（機能に影響なし）

### 出力フォーマット

```
## fix-review-comments 結果

### ✅ 適用した修正（N件）
1. [Agent X] {コメント内容}
   → 修正内容: {何を変更したか}

### ❌ 却下したコメント（N件）
1. [Agent X] {コメント内容}
   → 却下理由: {なぜ修正しないか}
```

## Phase 3: 動作確認

修正後に `npm run verify`（build + test）を実行して通過を確認する。

## 実行手順

1. `git diff main...HEAD` で変更差分を取得
2. 5エージェントを Agent ツールで **並列起動**（subagent_type: Explore）
3. 全エージェントの結果を収集・統合
4. **fix-review-comments フェーズ**: 各コメントを選別し、適切なものを修正
5. `npm run verify` を実行
6. 🔴 MUST が残っている場合は再修正 → verify のループ

## 最終出力フォーマット

```
## セルフレビュー結果

### Agent 1 — 規約番人 ✅ / ⚠️
- ...

### Agent 2 — セキュリティ監査 ✅ / ⚠️
- ...

### Agent 3 — 型・品質検査 ✅ / ⚠️
- ...

### Agent 4 — テスト検証 ✅ / ⚠️
- ...

### Agent 5 — 要件照合 ✅ / ⚠️
- ...

---

## fix-review-comments 結果

### ✅ 適用した修正（N件）
...

### ❌ 却下したコメント（N件）
...

---

## 最終判定

🔴 未解決の MUST:（あれば列挙）
🟡 未適用の SHOULD:（あれば列挙）

## 判定: ✅ PR 作成可 / ⚠️ 要修正
```

## 注意事項

- 5エージェント並列起動のためトークン消費が多い
- fix-review-comments は **意図的な設計を尊重する**。過剰修正しない
- 全 🔴 が解消されたら PR 作成に進む
