# タグ機能

## 概要

タスクにカンマ区切りのテキストでタグを付け、プロジェクト詳細画面でタグでフィルタリングできるようにする。

## ユーザーフロー

1. タスク作成・編集フォームに「タグ」入力欄を追加（例: `bug, urgent`）
2. タスク一覧でタグをバッジ表示
3. プロジェクト詳細画面にタグフィルターを追加 → 選択したタグに一致するタスクのみ表示

## 実装方針

### DBマイグレーション

- `tasks` テーブルに `tags text[] DEFAULT '{}'` カラムを追加（PostgreSQL 配列型）
- マイグレーションファイルを新規作成

### 変更・追加するファイル

| ファイル | 変更内容 |
|---|---|
| `supabase/migrations/xxx_add_tags_to_tasks.sql` | `tags` カラム追加 |
| `src/lib/supabase/queries.ts` | `createTask` / `updateTask` に tags 対応 |
| `src/app/projects/[id]/actions.ts` | `createTaskAction` に tags フィールド追加 |
| `src/app/projects/[id]/page.tsx` | タグフィルター UI + タスクカードにタグバッジ表示 |

### データ形式

- 入力: カンマ区切り文字列 `"bug, urgent"` → パース → `["bug", "urgent"]`
- 保存: `text[]`（PostgreSQL 配列）
- 表示: 各タグをバッジ（小さいチップ）として並べる

### フィルタリング

- プロジェクト詳細画面にそのプロジェクトのタグ一覧を表示
- クリックで絞り込み（クライアントサイドフィルタリングで十分）

## エッジケース

- タグなし（空配列）は正常ケース
- 前後の空白はトリムして保存（`"bug , urgent "` → `["bug", "urgent"]`）
- 重複タグは保存前に除去
- 1〜3個想定だが上限チェックは不要（UX優先）
